Git Vocab

Basic terms

Project = Repository (repo).

Working directory = local folder where project lives.

Staging = control what we are going to commmit.

Commit = Git's way of "saving" history of changes. Git stores it locally on the hard drive in a hidden system file. The project's files can be restored using git checkout -- .(space and a period) in the command line.

GitHub = remote repository.

Push&Pull = upload&download our commits to&from remote repository on GitHub.

Install Git.

Follow - Git's Homepage ( https://git-scm.com/ ). Click "Download".

Git personal settings:

First, open GitBash.

  • User name: git config --global user.name "Tom".
  • User e-mail: git config --global user.email "Tom@addres.com".

Basic commands.

pwd - print working directory.

cd dirName - change working directory. Can use "drag&drop" method. Print cd than space and drag the folder from the explorer window.

mkdir "name" - create directory.

git init - create new local repository in the folder.

touch "fileName" - create new file.

git status - current state of the working area.

git add -A - adding ALL files to the staging area.

git commit -m "Clear message." - saving changes.

git checkout -- . - restore to last commit.

git clone https://github.com/Agrotuss/travel-site.git - clone existing remote repository to local folder. Directory must be empty ( no git init ).

Creating GitHub repo

Follow your GitHub account (https://github.com/Agrotuss/).

Create new empty repository.

Create new folder on your computer. It name and the name of new repo must be the same.

Case A

Clone existing repo to your local folder:

git clone https://github.com/LearnWebCode/travel-site-files.git

You can check current repo url

git remote -v

Then change the remote repo url for your own

git remote set-url origin https://github.com/Agrotuss/udemy_template.git

Push local files to remote repo

git push origin master

Refresh your GitHub page. You'll see your files there.

Case B

You have create empty local repository

git init

Add remote repository url

git remote add origin https://github.com/Agrotuss/udemy_template.git

Then push files as in Case A.
